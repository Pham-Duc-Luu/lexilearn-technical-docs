# 📌 アーキテクチャの説明

上記の図は、システムがどのように動作するかを視覚的に示しています。アーキテクチャは大きく分けて **クライアント側** と **サーバー側** の 2 つのグループに分かれています。

## 🖥️ クライアント

- `frontend` は React（Vite）で開発されており、ユーザーインターフェースの表示を担当します。
- `apigateway` を通じてサーバーと通信します。
- クライアントに関心のある方は、[こちら](/docs/front-end-service/intro) をご覧ください。

## サーバー

`apigateway`（AWS API Gateway）：クライアントからの唯一のアクセス窓口であり、リクエストを他のサービスにルーティングします。

> マイクロサービスアーキテクチャは非常に広範です。そのため、各サービスに関するドキュメントは次のセクションでご確認ください。
